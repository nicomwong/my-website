UC Santa Barbara

Computer Engineering

Learning highlights
Distributed Systems
  • Mutexes: lock around a piece of data that more than one entity has access to
  • Logical clock protocol (Lamport): a server's perspective knows the relative ordering of events it has seen
  • Vector clock protocol: a server's perspective knows the relative ordering of events it has seen per server
  • Byzantine Fault-Tolerance: handling cases of server failure or corruption (i.e. it is hacked/compromised)
  • Paxos and Multi-Paxos Consensus protocols:
    • Purpose: Replicate a distributed state machine in a distributed system.
    • Consistency: Data consistency is gauranteed by the protocol
    • Fault-tolerance: as long as a majority of servers are alive, these properties are gauranteed
    • Scalability: Multi-Paxos is the most efficient version of Paxos Consensus. It limits the maximum number of rounds to 2 (?) per replication phase (which happens whenever the state machine is updated). The maximum number of rounds for the leader phase is 2(?).
    • Multi-Paxos vs. Paxos: Multi-Paxos eliminates the 3rd phase of Basic Paxos; Basic Paxos has 3 phases.
